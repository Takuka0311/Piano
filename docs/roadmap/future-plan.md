# 后续全量执行计划（M4+）

本文档汇总当前版本之后的所有主要工作，作为统一执行清单。

## 0. 当前完成基线
- M1：键位映射 + 曲谱调度 + CLI 闭环（已完成）。
- M2：WASAPI 最小发声、基础测试、CI 打包（已完成）。
- M3：持续混音、运行时健康检测与回退、CI 快慢分层（已完成/进行中收尾）。

---

## M4：轻量 GUI Alpha（第一版界面）

### 目标
- 提供最小可用桌面界面，不再只依赖命令行。
- 保持 UI 与音频/引擎解耦（命令队列通信）。

### 范围
- In Scope：
  - 主窗口与播放控制（Play/Stop）。
  - 键盘可视化按压反馈。
  - 文件选择（`*.keyboard` / `*.in`）。
  - 后端切换（`wasapi/log`）与基础参数配置。
- Out of Scope：
  - 高级皮肤、主题系统、复杂编辑器能力。

### 关键任务
1. 新增 `app_ui` 模块与最小窗口框架。
2. 建立 UI -> engine/audio 命令通道。
3. 把当前 CLI 参数转为 UI 配置项（并保留 CLI）。
4. 增加 UI 冒烟测试与最小手工回归清单。

### 验收
- 不使用命令行参数也能完成“加载曲谱并播放”。
- UI 与实际发声状态一致（播放中/停止/错误提示）。

---

## M5：兼容增强与稳定性强化

### 目标
- 对齐旧版关键行为，提升跨设备稳定性。

### 关键任务
1. DirectSound 回退后端接入。
2. 设备断开/切换自动恢复策略完善。
3. 配置持久化（上次后端、采样率、缓冲、最近文件）。
4. 错误码与用户可读错误提示统一。
5. 补齐 spec-002 的长时稳定性验证与结论。

### 验收
- 常见设备异常不崩溃，恢复或可回退。
- spec-002 状态可从 `review` 推进到 `implemented`（满足证据）。

---

## M6：乐器与 MIDI 能力补齐

### 目标
- 完整补齐 MIDI Out / VSTi 核心能力，增强演奏与兼容性。

### 关键任务
1. MIDI Out 设备枚举、选择、切换与错误处理。
2. VSTi 生命周期管理（加载、卸载、异常回收）。
3. 输出路径切换策略（WASAPI 合成 / MIDI Out / VSTi）。
4. 增加对应回归样例与脚本化验证。

### 验收
- 至少一条 VSTi 路径 + 一条 MIDI Out 路径稳定可用。
- spec-004 从 `draft` 推进到 `review/implemented`。

---

## M7：导出与可观测性

### 目标
- 增强工程可维护性与可交付能力。

### 关键任务
1. WAV 导出最小实现（离线渲染链）。
2. 性能/延迟指标采样与日志结构化。
3. 关键运行指标展示（缓冲占用、xruns/异常计数）。
4. Release 前自动化检查脚本增强（本地 + CI）。

### 验收
- 可导出可复现的 WAV 文件。
- 关键性能与异常可追踪。
