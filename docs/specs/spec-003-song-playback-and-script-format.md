# SPEC-003 曲谱回放与脚本格式

- 状态：`draft`
- 优先级：P0

## 背景与目标
- 新版本必须兼容旧版 `*.in` 曲谱语义，保证自动播放听感与控制行为一致。

## 范围
- In Scope：
  - `*.in` 读取与解析
  - 调度队列（开始/结束事件）
  - token 解释（音高、调式、速度、休止）
- Out of Scope：
  - 新脚本 DSL 扩展

## 数据定义
- 输入行：`begin id last db`
- 关键 token：
  - 音符：`1..7` + 可选 `#` + 八度 `+/-`
  - 控制：`A..G`、`a..g`、`S`、`0`

## 功能需求
- 解析器支持空行与注释（若定义）。
- 调度器按时间顺序正确触发 NoteOn/NoteOff。
- `S` 应更新拍长/速度参数并影响后续事件。
- 调式 token 应更新移调并作用于后续音符。

## 非功能需求
- 播放阶段不阻塞 UI 主线程。
- 长曲目播放不出现累计漂移明显放大。

## 验收标准
- 基线曲谱可完整播放。
- 与旧版相比，关键段落音高和节奏不偏离预期。
- 停止操作可及时终止后续事件并释放当前音符。

## 测试计划
- 单测：
  - token 解析；
  - 异常输入容错；
  - 时间差计算。
- 回归：
  - 基线曲谱 A/B 对比。

## 风险与回滚
- 风险：旧版隐式行为（例如边界值）未完全记录。
- 回滚：提供 `legacy_parser_mode` 按旧逻辑运行。
