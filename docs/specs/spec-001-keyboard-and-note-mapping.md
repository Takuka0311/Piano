# SPEC-001 键盘与音符映射

- 状态：`draft`
- 优先级：P0

## 背景与目标
- 新版本需完整兼容旧版键盘映射行为，支持 `*.keyboard` 文件驱动。
- 保证“按键按下/释放 -> 音符触发/释放”一致且可视化同步。

## 范围
- In Scope：
  - `*.keyboard` 解析
  - 按键事件映射到 MIDI key
  - UI 按压反馈
- Out of Scope：
  - 键盘映射编辑器高级功能（批量编辑、导入导出 UI）

## 功能需求
- 能读取文本映射：`<KeyName> <MidiKey>`
- 支持覆盖当前映射集（用于曲目播放模式）
- 支持按下/释放的去抖与重复键处理（忽略 auto-repeat 重复触发）
- 无映射键不触发发音

## 非功能需求
- 映射查找复杂度应为 O(1) 或近似 O(1)。
- 键位切换应在可感知即时范围内完成。

## 验收标准
- 给定基线映射文件，随机抽样 20 个键位，音高与旧版一致。
- 按住键不应重复触发 NoteOn。
- 释放后必须发出 NoteOff。

## 测试计划
- 单测：
  - 解析合法/非法行。
  - 重复键定义冲突策略。
- 集成测：
  - 输入事件到 MIDI 命令链路。

## 风险与回滚
- 风险：不同键盘布局/输入法导致 KeyName 不一致。
- 回滚：提供“legacy keyname mode”映射策略开关。
