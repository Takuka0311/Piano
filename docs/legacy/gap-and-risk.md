# 旧版缺口与风险分级

## P0（重构前必须正视）
- **时序精度风险**
  - 当前自动播放依赖主线程忙等待，受 UI 事件与系统调度影响明显。
- **模块耦合风险**
  - 播放、映射、UI、音频输出全部在 `Piano.cpp`，变更容易连锁回归。
- **可测试性风险**
  - 核心逻辑缺少独立可测层，无法做可靠自动化回归。
- **依赖风险**
  - 强依赖 `mdaPiano.dll` 与 VST2 风格接口，后续可维护性与兼容性受限。

## P1（第一阶段应改进）
- **错误处理不足**
  - 文件读取失败、格式错误、设备不可用等场景缺少统一错误策略。
- **配置硬编码**
  - 曲目名、采样率、缓冲大小、启动行为写死在代码内。
- **状态模型不清晰**
  - `lay/db/speed/isPlay/flag[]` 分散更新，不利于并发和可追踪性。

## P2（体验与工程质量提升项）
- **国际化与文本一致性**
  - 部分文案硬编码，缺少统一资源管理。
- **可扩展能力不足**
  - 缺少插件生命周期管理、设备热切换、更多脚本命令扩展点。

## 回归风险清单（重构阶段）
- `*.in` token 解释与旧版行为不一致（最容易造成功能“看似可用但听感错”）。
- 按键映射与 UI 反馈不同步。
- 复音释放时机与旧版差异导致粘音/断音。
- 速度控制 `S` 命令与拍长换算不兼容。

## 验收建议（旧版对齐）
- 基线曲目：至少用 `カワキヲアメク.in` 与 `梦中的婚礼.in` 作为兼容回归样本。
- 基线动作：手动按键、自动播放、暂停恢复、布局切换、力度变化。
- 基线指标：音高正确、节奏误差可控、无明显爆音/卡顿。
